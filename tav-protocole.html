Guide de réutilisation

Test d'Associations Verbales Informatisé (TAVI)

Ce protocole explique comment récupérer, déployer et personnaliser le TAVI pour vos propres recherches sur les représentations sociales.

GitHub Pages — gratuit Google Sheets — collecte auto Export CSV · Jamovi · SPSS · R 1 fichier HTML unique
Présentation

Qu'est-ce que le TAVI ?

Le TAVI est un outil de collecte de données en ligne pour l'étude des représentations sociales par la méthode des associations verbales. Il présente des termes inducteurs et recueille les associations libres des participants, puis envoie les données vers un Google Sheet pour analyse.

Pour quel usage ?

Toute recherche nécessitant l'exploration du noyau central d'une représentation sociale : croyances professionnelles, images de métiers, représentations d'objets éducatifs, de pratiques, d'institutions, etc.

Comment ça fonctionne ?

Un fichier HTML unique qui tourne dans n'importe quel navigateur. Le participant saisit ses associations, les données sont envoyées en temps réel vers votre Google Sheet via un script Apps.

Ce que vous allez modifier

Principalement le tableau inducteurs dans le code JavaScript et l'URL du script Google Apps. Vous pouvez aussi personnaliser les couleurs et les textes d'introduction.

⏱ Combien de temps ?

Comptez 30 minutes pour déployer votre propre version une fois ce protocole en main. Aucune connaissance avancée en programmation n'est nécessaire.

Protocole

Les étapes de réutilisation

Suivez ces 8 étapes dans l'ordre. Chaque étape est autonome et illustrée d'exemples concrets.

1
Récupération du code

Accéder au fichier source et copier le code

Rendez-vous sur la page de démonstration du TAVI à l'adresse suivante :

URL de la version de référence
https://perilordinaire.github.io/tav-raccrochage.html

Une fois sur la page, faites un clic droit → Afficher le code source (ou utilisez le raccourci Ctrl+U / Cmd+U). Cela ouvre le code source brut dans un nouvel onglet.

💡
Sur la page du code source, faites Ctrl+A puis Ctrl+C (ou Cmd+A / Cmd+C sur Mac) pour sélectionner et copier la totalité du code en une seule fois. Tout le TAVI tient dans un unique fichier HTML — c'est voulu pour faciliter la réutilisation.
2
Création de votre dépôt

Créer votre propre dépôt GitHub

Connectez-vous à GitHub (créez un compte gratuit si vous n'en avez pas). Cliquez sur le + en haut à droite → New repository.

Donnez-lui un nom explicite (ex. : mon-tav-recherche), cochez Public, cochez Add a README file, puis cliquez sur Create repository.

ℹ️
Le dépôt doit être public pour que GitHub Pages fonctionne gratuitement. Vos données de recherche seront collectées dans votre Google Sheet privé, pas exposées sur GitHub.
3
Création du fichier HTML

Coller le code dans votre dépôt

Dans votre nouveau dépôt, cliquez sur Add file → Create new file. Nommez le fichier exactement index.html, puis collez le code copié à l'Étape 1 dans la zone d'édition.

Cliquez sur Commit changes en bas de la page pour sauvegarder.

💡
Pour toutes les modifications futures, cliquez sur le fichier → icône crayon (Edit) → modifiez → Commit changes. La mise à jour est effective en moins d'une minute.
4
Personnalisation des inducteurs

Remplacer les termes inducteurs par les vôtres

Ouvrez votre fichier index.html en édition. Cherchez le tableau inducteurs avec Ctrl+F (il se trouve vers la ligne 690) :

Bloc à localiser et modifier
const inducteurs = [ "Décrochage scolaire", "Élève de lycée professionnel", "Raccrochage scolaire", "Compétences hors-école", "Parcours scolaire réussi" ];

Remplacez les termes par les vôtres en conservant la même structure (guillemets, virgules, crochets) :

Exemple — Vos propres termes
const inducteurs = [ "Motivation scolaire", "Apprentissage collaboratif", "Évaluation formative" ];
⚠️
Respectez impérativement la structure : chaque terme entre guillemets " ", séparé par une virgule, sans virgule après le dernier terme. Vous pouvez mettre entre 3 et 8 termes — au-delà, le taux d'abandon augmente significativement.
5
Configuration Google Sheets

Préparer la feuille de collecte et le script Apps

Les données saisies par vos participants seront automatiquement envoyées dans un Google Sheet que vous contrôlez. Cette étape se déroule en trois temps.

5.1 — Créer la feuille de calcul. Allez sur Google Sheets, créez une nouvelle feuille et nommez-la (ex. : "TAV — Données Recherche"). Dans la première ligne, copiez exactement ces en-têtes dans des cellules adjacentes :

En-têtes de colonnes — à copier exactement
Date/Heure | Sexe | Âge | Fonction | Discipline | Type Structure | Ancienneté Structure | Ancienneté Éducation | Diplôme | Support | Environnement | Isolement | Inducteur | Mot | Rang

5.2 — Créer le script Google Apps. Dans votre Google Sheet : Extensions → Apps Script. Supprimez le code par défaut et collez-y le code suivant :

Script Apps à coller
function doPost(e) { try { const data = JSON.parse(e.postData.contents); const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet(); data.data.forEach(row => { sheet.appendRow(row); }); return ContentService .createTextOutput(JSON.stringify({ 'status': 'success' })) .setMimeType(ContentService.MimeType.JSON); } catch (error) { return ContentService .createTextOutput(JSON.stringify({ 'status': 'error', 'message': error.toString() })) .setMimeType(ContentService.MimeType.JSON); } }

Enregistrez, puis cliquez sur Déployer → Nouveau déploiement. Sélectionnez Application Web. Configurez : Exécuter en tant que → Moi ; Qui a accès → Tout le monde. Cliquez sur Déployer et autorisez l'application.

⚠️
Copiez l'URL fournie à la fin du déploiement — elle ressemble à https://script.google.com/macros/s/AKfycby.../exec. Vous en aurez besoin à l'étape suivante. Si vous fermez la fenêtre sans la copier, vous devrez créer un nouveau déploiement.
6
Connexion du script

Intégrer l'URL du script dans votre code

Retournez dans votre index.html sur GitHub. Cherchez la ligne suivante (vers la ligne 870) avec Ctrl+F :

Ligne à localiser
const SCRIPT_URL = 'VOTRE_URL_GOOGLE_APPS_SCRIPT_ICI';

Remplacez le texte entre guillemets par votre URL de déploiement :

Après modification
const SCRIPT_URL = 'https://script.google.com/macros/s/VotreID/exec';

Sauvegardez en cliquant sur Commit changes.

ℹ️
Si vous modifiez le script Apps ultérieurement (correction d'un bug, par exemple), vous devrez créer un nouveau déploiement et mettre à jour cette URL dans le code. Un script modifié sans nouveau déploiement conserve l'ancienne version en production.
7
Mise en ligne

Activer GitHub Pages et tester le TAVI

7.1 — Activer Pages. Dans votre dépôt GitHub, allez dans Settings (en haut) → Pages dans le menu de gauche. Sous Source, sélectionnez la branche main et cliquez sur Save. Attendez 2–3 minutes.

Votre TAVI est en ligne à l'URL :

Votre URL publique
https://votre-username.github.io/nom-du-depot/

7.2 — Tester. Ouvrez l'URL et faites un test complet du début à la fin, puis vérifiez que les données arrivent bien dans votre Google Sheet.

  • L'interface s'affiche correctement et vos termes inducteurs apparaissent.
  • Le formulaire de données démographiques fonctionne.
  • La saisie des associations est fluide pour chaque terme.
  • Les données arrivent bien dans le Google Sheet après soumission.
  • Les colonnes correspondent aux en-têtes définis à l'Étape 5.1.
⚠️
Si les données n'arrivent pas dans le Google Sheet, vérifiez : (1) que vous avez bien autorisé le script Apps lors du déploiement, (2) que l'URL dans le code est correcte, (3) que les en-têtes du Google Sheet correspondent exactement au modèle.
8
Personnalisation optionnelle & attribution

Adapter l'apparence et conserver les références

Le TAVI est fonctionnel tel quel. Vous pouvez néanmoins ajuster son apparence selon votre contexte de recherche.

Les couleurs (cherchez :root vers la ligne 15) :

Variables de couleur
:root { --primary: #1a4d2e; /* Couleur principale */ --accent: #e8ab5e; /* Couleur d'accentuation */ }

Le titre (cherchez <h1> vers la ligne 580) et le texte d'introduction (vers la ligne 625) peuvent être personnalisés selon votre objet de recherche.

⚖️
Conservez impérativement les références méthodologiques dans le pied de page de l'application (ligne ~905) : Test d'Associations Verbales (TAV) d'après Abric (2003) et Vergès (1992) — Lamolet (2026). Utilisation des données collectées

Analyser les données dans Jamovi, SPSS ou R

Le TAVI produit un flux de données structurées vers votre Google Sheet. Voici les repères méthodologiques pour dimensionner votre collecte et analyser vos résultats.

📊 Dans Jamovi

Exportez votre Google Sheet en CSV. Importez-le dans Jamovi (Fichier → Ouvrir). Utilisez Analyses → Fréquences pour les tris à plat et calculez le rang moyen par terme pour identifier le noyau central.

📐 Dans SPSS ou R

Importez le CSV directement. Dans R, les packages FactoMineR et explor permettent des analyses prototypiques avancées. Calculez : fréquence d'évocation et rang moyen pour chaque association.

🎯 Noyau central

Croisez fréquence élevée et rang faible : les mots qui apparaissent souvent et en premier constituent le noyau central de la représentation (Vergès, 1992).

💾 Sauvegarde longue durée

Exportez régulièrement votre Google Sheet en CSV et conservez-le. Le Google Sheet peut être supprimé ou devenir inaccessible ; le CSV local est votre seule sauvegarde pérenne.

Paramètre Recommandation Justification
N 80 participants minimum ; 100–150 pour comparaisons robustes Abric (2003) — seuil de saturation prototypique
Inducteurs 3–5 termes (court, ~5–6 min) ; 6–8 termes (~8–10 min) Au-delà de 8, le taux d'abandon augmente significativement
Export CSV depuis Google Sheets Compatible Jamovi, SPSS, R, Excel directement
